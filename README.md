# Widget Controls Studio

Vue 3 + TypeScript студия для редактирования конфигов «widget block» — внутреннего формата, которым страничный билдер описывает переиспользуемые блоки (FAQ, таблицы, аккордеоны и подобные секции страницы) вместе со схемой их полей и форм редактирования.

## Идея

Каждый такой блок — это текстовый конфиг: набор полей (`content`/`style`) плюс словарь `widgets_controls`, который описывает, какая форма редактирования должна появиться в билдере для каждого поля — текстовое поле, цвет, файл, коллекция, переключатель и так далее, со своим типом валидации и разрешением (десктоп/планшет/телефон). Готовить и поддерживать этот словарь вручную — рутина, подверженная ошибкам: поля и контролы рассинхронизируются, забываются breakpoint-варианты, типы валидации не соответствуют реальным данным.

Widget Controls Studio закрывает именно эту рутину. Инженер вставляет сырой блок, студия разбирает его в структурированную модель, достраивает недостающие контролы по эвристикам, показывает всё в табличном и inspector-виде для правки, прогоняет диагностику и аудит, а на выходе отдаёт текст в том же формате, который ожидает платформа — так что цикл «вставил → поправил → вставил обратно» остаётся round-trip.

## Возможности

### Редактирование блока

- Разбор сырого блока в модель (обязательные поля, контент, произвольные style-группы, отдельная карта контролов) и обратная сборка в исходный текстовый формат.
- Табличный редактор параметров по секциям с drag-and-drop сортировкой и множественным выделением.
- Дерево структуры блока слева со схлопывающимися секциями (аккордеон) и счётчиком параметров в каждой.
- Правая панель — инспектор поля и инспектор контрола: группа, тип, разрешение, валидация, подсказка, live-предпросмотр самого контрола (текст, цвет, файл, коллекция, свитч и т.д.), сводка по параметру (раздел, источник, порядок).
- Четыре режима просмотра блока — исходный текст, разобранный объект, YAML контролов, построчный diff — через Monaco Editor, с полноэкранным режимом (иконка разворачивания, `Esc` для выхода).

### Контролы и автоматика

- Генератор недостающих контролов по эвристикам (тип поля → тип контрола → дефолтная валидация).
- Сценарии восстановления («быстрые исправления»): числовые контролы без валидации, файловые без конфига, контролы с неверной жирностью шрифта, рассинхронизированные группы, контролы-сироты, пересборка названий.
- Пресеты контролов и параметров — сохранить текущую настройку и применить её повторно (в режиме слияния или замены) к одному или сразу нескольким выбранным полям.
- Массовое редактирование: применение пресета, смена группы или breakpoint, генерация контролов — сразу по выборке полей.
- Инструменты рефакторинга: переименование поля/группы, перенос полей между секциями, создание breakpoint-вариантов.
- Автогенерация русских названий контролов по ключу поля.

### Контроль качества

- Диагностика в реальном времени (парсинг, контролы, валидация, нормализация, экспорт) с быстрыми фиксами по клику.
- Режим аудита с фильтрами (пропущенные контролы, сироты, числовые без валидации, файлы без конфига, разночтения в именовании, пропуски breakpoint'ов).
- Режим сравнения — импорт второго блока и построчное сопоставление полей/контролов с выборочным или полным копированием отличий.

### История и интерфейс

- Полная транзакционная история изменений (undo/redo) со снапшотами до/после и отдельной панелью истории.
- Импорт/экспорт: вставка сырого текста, копирование блока/контролов/JSON-объекта, скачивание файлов.
- Переключение тёмной/светлой темы, панели произвольной ширины (resize handles).
- Полностью русский интерфейс.

Горячие клавиши:

| Сочетание | Действие |
|---|---|
| `Ctrl S` | Экспорт |
| `Ctrl E` | Восстановить контролы |
| `Ctrl Shift N` | Нормализовать |
| `Ctrl Z` | Отменить |
| `Ctrl Shift Z` | Повторить |
| `Ctrl /` | Диагностика |

## Архитектура

Проект разложен на три слоя:

- **UI** (`src/components/studio/*`, `src/pages/WidgetStudioPage.vue`) — панели и модалки, в основном тонкие обёртки над стором.
- **Оркестрация** (`src/stores/*`, Pinia) — `widgetBlockStore` (модель блока, парсинг/экспорт, bulk-операции, сортировка), `editorStore` (история, черновик, layout, тема), `widgetStudioStore` (сценарии и пресеты), `selectionStore`, `auditStore`, `compareStore`, `presetsStore`.
- **Домен** (`src/modules/*`) — независимые от Vue модули с бизнес-логикой: парсер/нормализатор/сериализатор блока, генератор и восстановление контролов, движок диагностики и аудита, сравнение блоков, массовое редактирование, транзакционная история, пресеты, рефакторинг, сценарии восстановления, движок русских названий.

### Карта модулей

| Область | Расположение |
|---|---|
| Парсер блока | `modules/widget-block/parser/` |
| Нормализатор | `modules/widget-block/normalizer/` |
| Сериализатор / экспорт | `modules/widget-block/serializer/export-engine.ts` |
| Генератор контролов | `modules/controls/generator/` |
| Восстановление контролов | `modules/controls/repair/` |
| Диагностика | `modules/widget-block/validators/` + `modules/audit/` |
| Сравнение блоков | `modules/block-compare/` |
| Массовое редактирование | `modules/bulk-edit/services/` |
| История (undo/redo) | `modules/history/` |
| Пресеты | `modules/control-presets/`, `modules/parameter-presets/` |
| Рефакторинг | `modules/refactor/services/` |
| Сценарии восстановления | `modules/repair-scenarios/scenarios/` |
| Русские названия контролов | `modules/control-labels/` |

## Технологический стек

| Категория | Технологии |
|---|---|
| Фреймворк | Vue 3 (`<script setup lang="ts">` во всех компонентах), Vue Router |
| Состояние | Pinia |
| Сборка | Vite |
| Язык | TypeScript |
| Стили | Tailwind CSS v4 (CSS-first `@theme`, без `tailwind.config.js`) + кастомная система дизайн-токенов на CSS custom properties |
| Шрифты | Space Grotesk, IBM Plex Sans, IBM Plex Mono — self-hosted через `@fontsource` |
| Редактор кода | Monaco Editor — просмотр исходника/объекта/YAML/diff, полноэкранный режим |
| Данные | js-yaml — сериализация YAML-представления контролов |
| Тесты | Vitest + `@vue/test-utils`, `happy-dom` |
| Линт / формат | ESLint (`typescript-eslint`, `eslint-plugin-vue`) + Prettier |

## Дизайн-система

Внутреннее название темы — «Nocturne/Brass»: тёмная чернильно-фиолетовая база (`ink` / `panel` / `panel-elevated`) с двумя акцентами — violet для действий и brass/signal (тёплый охристый) для состояний и сгенерированных значений. Панели плоские, без теней и градиентов, разделены тонкими hairline-границами. Поддерживается светлая тема — переключение мгновенное, все цвета реактивны через CSS custom properties, на которые ссылаются сгенерированные Tailwind-утилиты.

## Быстрый старт

```bash
yarn install
yarn dev        # dev-сервер (Vite)
yarn build      # vue-tsc -b && vite build — типы проверяются перед сборкой
yarn preview    # локальный просмотр production-сборки
yarn test       # vitest (watch)
yarn test:run   # vitest (одноразовый прогон)
yarn lint       # eslint .
yarn format     # prettier --write .
```

## Тестирование

Модульные тесты (Vitest) покрывают доменную логику: парсер и нормализатор блока, генератор контролов, движок диагностики и аудита, сравнение блоков, массовое редактирование, пресеты, рефакторинг, сценарии восстановления, транзакционную историю. Компонентных и интеграционных тестов пока нет — это осознанный пробел, а не недосмотр.

## Известные ограничения

- Экспорт детерминирован, но не сохраняет исходное форматирование и комментарии — блок всегда пересобирается из модели.
- Автосохранение — черновик в `localStorage` с debounce, а не полноценная персистентность рабочего пространства; менеджера локальных проектов нет.
- Нет глобального поиска и виртуализации таблиц — на очень больших блоках возможна просадка производительности.
- Импорт поддерживает только вставку текста, без загрузки файла или слияния с текущим блоком.
